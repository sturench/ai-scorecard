# AI Reality Check Scorecard - Product Requirements Document

## Executive Summary

The AI Reality Check Scorecard is a strategic lead generation tool designed for C-suite executives to evaluate their organization's AI readiness and implementation effectiveness. It serves as the primary funnel for Stuart's AI Reality Check Specialist consulting services, targeting decision-makers who need guidance on AI strategy and implementation.

**Core Value Proposition:** A 10-minute assessment that provides immediate, actionable insights into AI readiness while qualifying and nurturing high-value consulting leads.

## Business Objectives & Success Metrics

### Primary Goals
- Generate 50+ qualified leads per month within 6 months
- Achieve >15% conversion rate from assessment completion to Executive Briefing consultation
- Build database of 1000+ C-suite contacts for ongoing nurturing
- Create predictable consulting pipeline worth $50k+ monthly within 12 months

### Key Performance Indicators
- **Assessment Completion Rate:** >70% of starters finish full assessment
- **Time to Complete:** 8-10 minutes actual vs. 10-minute advertised
- **Email Capture Rate:** >50% provide email during assessment
- **Executive Briefing Conversion:** >15% schedule consultation
- **Page Load Time:** <2 seconds for all pages
- **API Success Rate:** >98% for HubSpot integration

**Reference:** See `implementation-spec/business-requirements.md` for complete business objectives, target market analysis, and financial projections.

### Primary Personas

**Market Opportunity:** 24,000-36,000 target executives in companies actively implementing AI and seeking external guidance.

## Product Overview

### Core Assessment Structure
- **Format:** Sequential 4-step flow (not single page or adaptive)
- **Questions:** 16 total questions (3-4 per assessment area)
- **Duration:** 8-10 minute target completion time
- **Assessment Areas with Weights:**
  - AI Value Assurance (25%) - ROI measurement, spend controls, KPI tracking
  - Customer-Safe AI (35%) - Reliability, accuracy monitoring, failure handling
  - Model Risk & Compliance (25%) - Privacy compliance, bias testing, audit trails
  - Implementation Governance (15%) - QA processes, version control, monitoring

### Scoring & Results System
- **Scoring Range:** 0-100 overall score with 5 categories:
  - 80-100: "AI Reality Champion" (Excellence with mature processes)
  - 60-79: "AI Value Builder" (Good foundation, specific gaps)
  - 40-59: "AI Risk Zone" (Significant issues preventing value)
  - 20-39: "AI Theater Alert" (Creating more risk than value)
  - 0-19: "AI Crisis Mode" (Immediate intervention needed)

**Reference:** See `implementation-spec/assessment-content.md` for complete question sets, scoring methodology, and recommendation frameworks.

### Lead Capture & CRM Integration
- **Strategy:** Progressive data capture throughout assessment (optional email → enhanced details → executive briefing qualification)
- **HubSpot Integration:** Real-time contact creation with custom AI assessment properties
- **Lead Qualification Tiers:**
  - Tier 1 (Basic): Email provided → HubSpot contact creation
  - Tier 2 (Enhanced): Name + Company → Enhanced contact record
  - Tier 3 (Executive Briefing): Complete contact OR score <60 → Deal pipeline creation

**Reference:** See `implementation-spec/business-requirements.md` sections on CRM integration and lead qualification logic.

## Technical Architecture

### Technology Stack
- **Framework:** Next.js 14.2.x with App Router, React 18.3.x, TypeScript 5.5.x
- **Database:** PostgreSQL with Prisma ORM
- **Styling:** Tailwind CSS 3.4.x with CSS custom properties for theming
- **CRM Integration:** HubSpot Free CRM (100 requests per 10 seconds rate limit)
- **Email Service:** Resend or SendGrid with React Email templates
- **Deployment:** Vercel (recommended) with auto-scaling

### Core System Components
1. **Assessment Engine** - Sequential step flow with progress persistence
2. **Lead Capture System** - Progressive data collection with HubSpot sync
3. **Results Delivery System** - Interactive results page with email automation
4. **CRM Integration** - Real-time HubSpot sync with retry queue for resilience

**Reference:** See `implementation-spec/technical-architecture.md` for complete component hierarchy, implementation patterns, and performance optimization strategies.

### Database Design
- **Primary Entity:** Assessment table with responses, scores, user data, and HubSpot sync status
- **Session Management:** 24-hour sessions for incomplete assessments
- **Privacy Compliance:** Auto-scrub emails after 30 days, anonymous analytics retention
- **Resilience:** HubSpot sync queue for failed API calls with exponential backoff

**Reference:** See `implementation-spec/database-design.md` for complete schema, relationships, and data management policies.

## User Experience & Flow

### Assessment Flow
1. **Landing Page** → Clear value proposition, start assessment CTA
2. **Optional Email Capture** → Progressive enhancement, not mandatory
3. **Step 1-4: Assessment Questions** → 4 questions per step, progress indicators
4. **Results Display** → Score visualization, personalized recommendations
5. **Progressive Contact Capture** → Name/company → Phone → Executive Briefing qualification
6. **Email Delivery** → Automated results email with follow-up sequences

### Mobile-First Design
- **Mobile Optimization:** Support 40%+ mobile executive users
- **Responsive Design:** Equivalent performance across all devices
- **Touch-Friendly:** Large touch targets, swipe navigation

**Reference:** See `implementation-spec/user-experience-flow.md` for complete wireframes, user journey maps, and interaction patterns.

## API Specifications

### Core Endpoints
- `POST /api/assessment/start` - Create new assessment session
- `POST /api/assessment/save-progress` - Save step responses and optional email
- `POST /api/assessment/submit` - Complete assessment and trigger scoring
- `GET /api/assessment/results/[id]` - Retrieve assessment results
- `POST /api/hubspot/contact` - Create or update HubSpot contact
- `POST /api/email/send` - Send assessment results email

### Data Contracts
- Request/response validation with Zod schemas
- Error handling with consistent error response format
- Rate limiting (10 assessment starts per hour per IP)
- Session validation for all assessment operations

**Reference:** See `implementation-spec/api-specification.md` for complete endpoint documentation, request/response examples, and error handling specifications.

## Content Requirements

### Assessment Content
- **Question Language:** CEO/COO appropriate, scenario-based where possible
